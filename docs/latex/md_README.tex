\DoxyHorRuler{0}


\doxysubsubsection*{üéØ Purpose}

This example demonstrates the {\bfseries{basic behavior of the {\ttfamily fork()} system call}} and how a process is {\bfseries{duplicated}} into a parent and a child process. 

The goal is to understand\+: 


\begin{DoxyItemize}
\item how to distinguish parent and child processes 
\item how execution flow is duplicated 
\item how concurrent processes produce output 
\end{DoxyItemize}

\DoxyHorRuler{0}


\doxysubsubsection*{üõ†Ô∏è Description}

The program executes a decreasing {\ttfamily for} loop ({\ttfamily i = 2 ‚Üí 1}). At each iteration\+: 


\begin{DoxyEnumerate}
\item the {\ttfamily fork()} system call is invoked 
\item the {\bfseries{parent process}} receives a value greater than {\ttfamily 0} 
\item the {\bfseries{child process}} receives {\ttfamily 0} 
\item based on the return value\+: 
\begin{DoxyItemize}
\item the parent prints {\ttfamily i} 
\item the child prints {\ttfamily -\/i} 
\end{DoxyItemize}
\end{DoxyEnumerate}

Since {\ttfamily fork()} duplicates the current process\+: 


\begin{DoxyItemize}
\item after the first iteration there are {\bfseries{2 running processes}} 
\item after the second iteration there are {\bfseries{4 running processes}} 
\end{DoxyItemize}

The order of the printed output is {\bfseries{non-\/deterministic}}, because it depends on the operating system scheduler.
