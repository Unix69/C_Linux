72  A set of <b>C examples</b> introducing to Unix filesystem, mulitprocess and multithread programming, using certain <b>C standard functions</b> and <b>C Unix-like system calls</b>, related the below mentioned <b>headers</b>.
73  </div>
74  <br>
75  <ul style="list-style-type: none;">
76  <li><b>C Standard headers: </b> <code>string.h</code>, <code>stdio.h</code>, <code>stdlib.h</code>, <code>assert.h</code>, <code>signal.h </code>, <code>sys/stat.h</code></li>
77  <li><b>C Unix-like headers: </b> <code>fcntl.h</code>, <code>unistd.h</code>, <code>sys/types.h</code>, <code>pthread.h</code></li>
78  </ul>
79  <br>
80  <div style="font-size: 14px; margin-top: 0;">For example the usage of <b>string.h</b> functions such as <code>strlen</code>, <code>strcpy</code>, <code>strcmp</code>, or Unix system calls related to processes, filesystem and threads such as:
81  <ul>
82  <li><code>execl</code>, <code>execlp</code>, <code>execv</code>, <code>system</code>, <code>fork</code></li>
83  <li><code>signal</code>, <code>wait</code>, <code>waitpid</code>, <code>pause</code>, <code>kill</code></li>
84  <li><code>pipe</code>, <code>opendir</code>, <code>readdir</code>, <code>mkdir</code>, <code>closedir</code>, <code>open</code>, <code>read</code>, <code>write</code>, <code>close</code></li>

137  <h2>‚öôÔ∏è Make Project</h2>
138  <ol>
139  <li> <b>Install</b> <b>make</b>, <b>doxygen</b> and <b>gcc</b> tools if needed.</li>
140  <li> <b>Open a terminal</b> in <code>./</code>.</li>
141  <li>
142  <ul>
143  <li> A. Run make as <code>make</code> to build and generate C example's executables and C_Linux HTML Documentation using one-shot command.
144  <li> B. Run make as<ul>
145  <li> <code>make build</code> to build the .o object files of each C example.</li>
146  <li> <code>make build_doc</code> to build the documentation resources to produce it. </li>
147  <li> <code>make doc</code> to generate the doxygen HTML documentation starting from documentation resources</li>
148  <li> <code>make link_all</code> to link for each C example, all .o object file contained into an executable file.</li>
149  </ul>
150  </li>
151  <li> C. Run gcc on each C example you want to build, then generate the documentation with <code>make doc_build</code> and <code>make doc</code><ul>
152  <li> Tipically <code>gcc -c C-Unix-xxx/main.c -o main.o</code> to build the .o object files of the given example.</li>
153  <li> Then <code>gcc -c C-Unix-xxx/main.o -o program</code> to generate the executable named <code>program</code>.</li>
154  <li> <code>make doc_build</code> to build the documentation resources to produce it.</li>
155  <li> <code>make doc</code> to generate the doxygen HTML documentation starting from documentation resources.</li>
156  </ul>
157  </li>
158  </ul>
159  </li>
160  </ol>
161  <br>
162  <ul style="list-style-type: none;">
163  <li> ‚úÖ Example executables and .o object files generated into direct sub-directories of <code>src/</code>.</li>
164  <li> ‚úÖ Project Documentation generated at <code>docs/</code>. You can find Doxygen HTML documentation at <code>docs/html/index.html</code></li>
165  <li> ‚ö†Ô∏è Documentation Generation Warnings logged to <code>docs/doxygen_warnings.log</code></li>
166  <li> ‚ö†Ô∏è On Windows you can use the default make toolchain, but the build process will provide you <b>error</b> and <b>warning</b> on <b>certain examples</b>. So be carefully on running the generated executables.</li>
167  </ul>
