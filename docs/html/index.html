A set of C examples showing usage of certain C standard functions and C Unix-like system calls, exploring the below mentioned header files.

  • C Standard headers supported on Linux and Windows: string.h, stdio.h, stdlib.h, assert.h, signal.h , sys/stat.h
  • C Unix-like headers supported only on Unix based systems: fcntl.h, unistd.h, sys/types.h, pthread.h

For example the usage of string.h functions such as strlen, strcpy, strcmp, or Unix system calls related to processes, filesystem and threads such as:
  • execl, execlp, execv, system, fork
  • signal, wait, waitpid, pause, kill
  • pipe, opendir, readdir, mkdir, closedir, open, read, write, close
  • pthread_create, pthread_join

‚öôÔ∏è Make Project

  1. Install make, doxygen and gcc tools if needed.
  2. Open a terminal in ./.
    • A. Run make as make to build and generate C example's executables and C_Linux HTML Documentation using one-shot command.
    • B. Run make as
      • make build to build the .o object files of each C example.
      • make build_doc to build the documentation resources to produce it.
      • make doc to generate the doxygen HTML documentation starting from documentation resources
      • make link_all to link for each C example, all .o object file contained into an executable file.
    • C. Run gcc on each C example you want to build, then generate the documentation with make doc_build and make doc
      • Tipically gcc -c C-Unix-xxx/main.c -o main.o to build the .o object files of the given example.
      • Then gcc -c C-Unix-xxx/main.o -o program to generate the executable named program.
      • make doc_build to build the documentation resources to produce it.
      • make doc to generate the doxygen HTML documentation starting from documentation resources.

  • ‚úÖ Example executables and .o object files generated into direct sub-directories of src/.
  • ‚úÖ Project Documentation generated at docs/. You can find Doxygen HTML documentation at docs/html/index.html
  • ‚ö†Ô∏è Documentation Generation Warnings logged to docs/doxygen_warnings.log
  • ‚ö†Ô∏è On Windows you can use the default make toolchain, but the build process will provide you error and warning on certain examples. So be carefully on running the generated executables.
